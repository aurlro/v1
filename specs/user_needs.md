# Besoins Utilisateur

## Problématique Initiale
- Comprendre l’accusation d’« ego » dans des disputes à chaud et décoder les schémas émotionnels sous-jacents.
- Réduire la spirale défensive/sauveur/martyr et sécuriser l’état mental immédiat (logement, sommeil, sécurité).

## Objectifs Prioritaires
1. **Survivre à la crise** : disposer d’un bouton « pause » et de rappels opérationnels pour désamorcer en temps réel.
2. **Analyser à froid** : rejouer l’échange, identifier l’ego dominant et concevoir une réponse MVP alignée sur le besoin réel.
3. **Capitaliser** : conserver un journal horodaté, comparer les récurrences (types d’ego, fréquence) et mesurer la progression.
4. **S’entraîner** : répéter des scénarios en environnement sûr (Dojo) pour automatiser les validations émotionnelles.
5. **Être assisté** : déléguer l’analyse d’un message complexe à une IA briefée avec le prompt Persona.

## Persona de Référence
- **Utilisateur** : Product Owner/Business Analyst, habitué à la terminologie produit (bug, backlog, MVP).
- **Approche attendue** : langage analytique, livrables actionnables, plusieurs options de réponse avec objectif clair (désescalade, limite, alignement).
- **Contraintes** : confidentialité totale (outil local), disponibilité hors ligne, absence de solution miracle ; l’accompagnement doit rester un guide.

## Critères de Satisfaction
- Temps de préparation < 5 minutes pour compléter une analyse manuelle.
- Capacité à retrouver un script passé en < 15 secondes via le journal filtré.
- Export JSON en un clic pour sécuriser les données avant changement d’appareil.
